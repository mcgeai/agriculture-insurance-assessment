# 农业保险IT员工通用胜任力综合测评网站 — 需求文档

## 1. 项目概述

### 1.1 项目背景

农业保险公司IT部门承担着核心业务系统运维、数字化产品开发、数据安全保障等关键职责。IT员工不仅需要具备扎实的通用IT技能，还需理解农业保险业务逻辑、遵循行业合规要求。目前缺乏针对该复合型岗位的系统性胜任力评估工具，导致人才画像模糊、培训方向不清晰。

本项目旨在构建一套在线胜任力测评系统，为农业保险公司IT部门提供标准化、可量化的员工能力评估方案。

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| 标准化评估 | 建立统一的4维度胜任力模型，消除主观评价偏差 |
| 数据驱动 | 积累历史测评数据，为人才发展决策提供依据 |
| 持续改进 | 通过改进建议闭环，驱动员工能力提升 |
| 管理透明 | 管理层可查看全员测评数据，掌握团队能力分布 |

### 1.3 目标用户

| 角色 | 描述 |
|------|------|
| 普通员工（IT部门） | 参加在线测评，查看个人报告和历史记录 |
| 管理员（HR/部门主管） | 管理题库、查看全员测评数据、导出报表 |

## 2. 功能需求

### 2.1 用户认证模块

#### FR-AUTH-001 员工登录

- **输入**：工号 + 密码
- **规则**：
  - 工号为公司分配的唯一标识，6-10位数字
  - 密码长度≥8位，需包含字母和数字
  - 连续5次登录失败，账户锁定30分钟
- **输出**：登录成功后签发JWT Token，跳转至测评首页

#### FR-AUTH-002 管理员登录

- 管理员通过工号+密码登录，系统根据角色字段区分权限
- 管理员登录后跳转至管理后台

#### FR-AUTH-003 密码修改

- 首次登录强制修改密码
- 支持后续主动修改密码，需验证旧密码

### 2.2 测评首页模块

#### FR-HOME-001 胜任力维度展示

首页以卡片形式展示4个胜任力维度：

| 序号 | 维度 | 说明 |
|------|------|------|
| D1 | 农业保险专业知识 | 农险条款、承保理赔流程、农业灾害知识、政策性农险体系 |
| D2 | 系统操作能力 | 核心业务系统操作、数据查询与报表、系统集成与接口、故障排查 |
| D3 | 问题解决与沟通协调 | 需求分析与转化、跨部门协作、用户服务意识、应急响应 |
| D4 | 合规与风险意识 | 数据安全与隐私保护、监管合规要求、操作风险防控、审计配合 |

- 每张卡片显示：维度名称、维度说明、题目数量(5题)、当前状态（未测评/已完成）
- 提供「开始测评」按钮，仅当所有维度均为"未测评"时可点击

#### FR-HOME-002 测评入口控制

- 每次测评必须完成全部4个维度（共20题）后统一提交，不允许部分提交
- 提交后该次测评结束，无法修改答案
- 可发起新一轮测评，历史记录保留

### 2.3 答题模块

#### FR-QUIZ-001 答题界面

- 按维度顺序逐维度作答，每个维度5道单选题
- 每题4个选项（A/B/C/D），必须选择一项才能进入下一题
- 支持上一题/下一题导航
- 答题进度条显示当前维度完成度（如 3/5）
- 全部20题完成后显示「提交测评」按钮

#### FR-QUIZ-002 题目内容规范

每个维度的5道题覆盖该维度下的核心知识点，示例：

**D1 农业保险专业知识**
- 农险条款与保险责任
- 承保流程与验标要求
- 理赔查勘与定损方法
- 政策性农险补贴机制
- 农业灾害类型与影响评估

**D2 系统操作能力**
- 核心业务系统功能模块
- 承保/理赔系统操作流程
- 数据查询与报表生成
- 系统间数据对接
- 常见故障排查步骤

**D3 问题解决与沟通协调**
- 业务需求分析与IT方案转化
- 跨部门协作与信息同步
- 用户问题响应与处理
- 应急事件处置流程
- 文档编写与知识沉淀

**D4 合规与风险意识**
- 个人信息保护法要求
- 数据分级与访问控制
- 监管报送与合规审计
- 操作风险识别与报告
- 变更管理与审批流程

#### FR-QUIZ-003 答题计时

- 测评开始后计时，总时长限制60分钟
- 剩余5分钟时弹窗提醒
- 超时自动提交当前答案

### 2.4 评分与报告模块

#### FR-SCORE-001 评分规则

- 每题答对得1分，答错得0分
- 各维度得分 = (该维度答对题数 / 5) × 100，取整数
- 总分 = (总答对题数 / 20) × 100，取整数

#### FR-SCORE-002 胜任力评级

| 总分区间 | 评级 | 说明 |
|----------|------|------|
| 90-100 | A（卓越） | 全面达标，可承担核心岗位 |
| 75-89 | B（熟练） | 基本达标，个别维度需提升 |
| 60-74 | C（合格） | 勉强达标，需系统性培训 |
| 0-59 | D（待提升） | 不达标，需重点辅导 |

#### FR-SCORE-003 测评报告页面

报告内容包含：

1. **基本信息**：姓名、工号、部门、测评时间
2. **总得分与评级**：醒目展示总分和评级
3. **雷达图**：ECharts雷达图展示4个维度得分，直观反映能力分布
4. **各维度详情**：
   - 维度名称 + 得分
   - 该维度答对题数/总题数
   - 维度评级（同总分评级规则，阈值乘以5即每维度的题数阈值）
5. **改进建议**：根据薄弱维度给出针对性建议

##### 改进建议映射表

| 薄弱维度 | 改进建议 |
|----------|----------|
| D1 农业保险专业知识 | 建议参加农险业务培训，学习承保理赔实务，了解政策性农险最新政策文件 |
| D2 系统操作能力 | 建议加强核心系统实操训练，熟悉数据查询与报表功能，参与故障应急演练 |
| D3 问题解决与沟通协调 | 建议参与跨部门项目协作，提升需求分析能力，加强用户服务意识培训 |
| D4 合规与风险意识 | 建议学习《个人信息保护法》及公司数据安全制度，参加合规培训与案例学习 |

### 2.5 历史记录模块

#### FR-HIST-001 测评历史列表

- 按时间倒序展示所有历史测评记录
- 每条记录显示：测评时间、总分、评级、各维度得分摘要
- 点击记录可查看完整测评报告
- 支持对比两次测评结果（展示分数变化趋势）

### 2.6 管理员后台模块

#### FR-ADMIN-001 题目管理

- 题目列表：按维度筛选，支持分页
- 新增题目：选择维度 → 输入题干 → 输入4个选项 → 标记正确答案
- 编辑题目：修改题干、选项、正确答案
- 删除题目：二次确认后删除
- 题目启用/停用：停用的题目不出现在测评中

#### FR-ADMIN-002 测评数据总览

- 参评人数统计
- 全员平均分、最高分、最低分
- 各评级分布（饼图）
- 各维度平均分（柱状图）
- 支持按时间段筛选

#### FR-ADMIN-003 员工测评明细

- 员工列表：工号、姓名、部门、最近测评时间、最近评级
- 点击员工可查看其全部测评历史
- 支持按部门、评级筛选
- 支持导出Excel

#### FR-ADMIN-004 员工账号管理

- 新增员工账号（工号、姓名、部门、初始密码）
- 重置密码
- 启用/停用账号
- 设置管理员角色

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | ≤ 2秒 |
| API响应时间 | ≤ 500ms（常规接口） |
| 并发用户数 | 支持50人同时在线测评 |
| 数据库查询 | 单表查询 ≤ 100ms |

### 3.2 安全要求

| 项目 | 要求 |
|------|------|
| 认证 | JWT Token，有效期2小时，支持刷新 |
| 密码存储 | bcrypt加密，salt rounds ≥ 10 |
| 传输安全 | HTTPS |
| XSS防护 | 前端输入转义，后端输出编码 |
| SQL注入 | 使用参数化查询 |
| 权限控制 | 基于角色的API访问控制 |

### 3.3 可用性要求

| 项目 | 要求 |
|------|------|
| 浏览器兼容 | Chrome 90+、Firefox 90+、Edge 90+ |
| 响应式 | 支持桌面端（1280px+），平板端适配（768px+） |
| 操作反馈 | 所有操作有加载状态提示，错误有明确提示信息 |

### 3.4 数据要求

| 项目 | 要求 |
|------|------|
| 数据备份 | 每日自动备份SQLite数据库文件 |
| 数据保留 | 测评记录永久保留 |
| 导出 | 支持导出测评数据为Excel格式 |

## 4. 数据模型概览

### 4.1 核心实体

```
User（用户）
├── id, employee_no, name, department, password_hash, role, status, created_at, updated_at

Question（题目）
├── id, dimension, content, option_a, option_b, option_c, option_d, correct_answer, is_active, created_at, updated_at

Assessment（测评记录）
├── id, user_id, total_score, rating, started_at, submitted_at

Answer（答题明细）
├── id, assessment_id, question_id, selected_answer, is_correct

DimensionScore（维度得分）
├── id, assessment_id, dimension, score, correct_count
```

### 4.2 实体关系

- User 1:N Assessment（一个员工多次测评）
- Assessment 1:N Answer（一次测评多条答题记录）
- Assessment 1:4 DimensionScore（一次测评4个维度得分）
- Question 1:N Answer（一道题可能被多次作答）

## 5. 业务规则汇总

| 编号 | 规则 |
|------|------|
| BR-001 | 测评必须完成全部20题才能提交 |
| BR-002 | 每次测评从每个维度随机抽取5道有效题目 |
| BR-003 | 提交后不可修改答案 |
| BR-004 | 测评限时60分钟，超时自动提交 |
| BR-005 | 连续5次登录失败锁定30分钟 |
| BR-006 | 首次登录强制修改密码 |
| BR-007 | 评级由系统自动计算，不可人为调整 |
| BR-008 | 停用的题目不参与测评抽题 |
